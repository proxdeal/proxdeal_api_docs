<Note>**Pricing:** Credit-based — see dashboard for current rates</Note>

```
POST https://api.proxdeal.com/v0/longlist/generate
```

Runs the ProxDeal agentic discovery pipeline. Given a free-text target profile and a mode (`buyside` or `sellside`), the pipeline performs semantic search followed by LLM-based match-fit analysis and returns a ranked list of relevant DACH companies with match scores and reasoning.

<Warning>
  This endpoint runs a multi-step AI pipeline and may take **up to 4 minutes** to complete. Set your client timeout to at least 300 seconds (5 minutes).
</Warning>

## Modes

| Mode | Use case |
|---|---|
| `buyside` | Find acquisition targets matching the description |
| `sellside` | Find potential buyers for a company — requires `company_name` |

## Request body

<ParamField body="description" type="string" required>
  Free-text description of the target company profile. Be specific about products, services, and industry niche. Minimum 20 characters.

  Example: `"German software company that provides ERP solutions for the construction industry."`
</ParamField>

<ParamField body="mode" type="string" required>
  Search mode: `"buyside"` or `"sellside"`.
</ParamField>

<ParamField body="company_name" type="string">
  Required when `mode="sellside"`. The legal name of the company being sold. Used to anchor the search.
</ParamField>

<ParamField body="industry" type="string">
  Industry or sector keyword to narrow the search (e.g. `"Medizintechnik"`, `"SaaS"`, `"Maschinenbau"`).
</ParamField>

<ParamField body="example_urls" type="string[]">
  Up to 10 reference company website URLs. Used to calibrate the semantic search — the pipeline uses these as positive examples.
</ParamField>

<ParamField body="states" type="string[]">
  Restrict results to specific German Bundesländer (e.g. `["Bayern", "Baden-Württemberg"]`).
</ParamField>

<ParamField body="wide_search" type="boolean" default="false">
  If `true`, broadens the match threshold to return more results (but with lower average precision).
</ParamField>

<ParamField body="followup_searches" type="boolean" default="true">
  If `true`, the pipeline automatically expands the search using initial results as anchors. Improves recall at the cost of additional runtime.
</ParamField>

## Example

<CodeGroup>
```bash cURL
curl -X POST https://api.proxdeal.com/v0/longlist/generate \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "description": "B2B SaaS company providing HR software for mid-sized German companies",
    "mode": "buyside",
    "industry": "HR Software",
    "states": ["Bayern", "Baden-Württemberg"]
  }'
```
```python Python

resp = httpx.post(

    },
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    timeout=300.0,

result = resp.json()
companies = result["data"]
```

## Response fields

<ResponseField name="data" type="object[]">
  Matched companies, sorted by `match_score` descending.

  <Expandable title="Longlist result">
    <ResponseField name="sdac_id" type="string | null">ProxDeal `company_id` — use with `GET /v0/company/{id}` for the full profile.</ResponseField>
    <ResponseField name="match_score" type="number | null">Semantic match score (0.0–1.0). Higher = better fit.</ResponseField>
    <ResponseField name="match_reasoning" type="string | null">LLM-generated explanation of why this company matches the profile.</ResponseField>
    <ResponseField name="company_legal_name" type="string | null">Official legal name.</ResponseField>
    <ResponseField name="company_legal_form" type="string | null">Legal form.</ResponseField>
    <ResponseField name="base_url" type="string | null">Company website domain.</ResponseField>
    <ResponseField name="description_bw_webscraped" type="string | null">AI description from website scrape.</ResponseField>
    <ResponseField name="description_hr_gegenstand" type="string | null">Business purpose from the commercial register.</ResponseField>
    <ResponseField name="industry" type="string | null">Hierarchical industry path.</ResponseField>
    <ResponseField name="type_label_ger" type="string | null">Business model label.</ResponseField>
    <ResponseField name="address_state" type="string | null">Bundesland / canton.</ResponseField>
    <ResponseField name="country_iso_code" type="string | null">Country code: `DE`, `AT`, or `CH`.</ResponseField>
    <ResponseField name="employees_class" type="string | null">Employee size class (e.g. `"10-49"`, `"50-249"`).</ResponseField>
    <ResponseField name="total_assets" type="number | null">Total assets from the most recent financial report (EUR).</ResponseField>

## Errors

| Status | Code | Cause |
|---|---|---|
| 401 | `AUTH_INVALID_KEY` | Invalid or missing API key |
| 402 | — | Insufficient credit balance |
| 422 | `INVALID_MODE` | `mode` is not `buyside` or `sellside` |
| 422 | `COMPANY_NAME_REQUIRED` | `company_name` missing when `mode=sellside` |
| 429 | `RATE_LIMIT_EXCEEDED` | Rate limit hit |
| 503 | — | Discovery pipeline unavailable |
