---
title: "Lookup by URL"
description: "Resolve a website URL to company records."
openapi: "GET /v0/lookup/url"
---

Resolves a website URL to matching company records by normalising the input to its bare domain and matching against the ProxDeal URL index. Protocol, `www.` prefix, paths, and query parameters are all stripped before matching. Ideal for CRM enrichment or browser extensions where you have a URL and need a `company_id`.

## URL normalisation

| Input | Normalised to |
|---|---|
| `https://www.example.com/about?ref=1` | `example.com` |
| `http://example.com` | `example.com` |
| `www.example.com` | `example.com` |
| `example.com` | `example.com` |

The response structure is identical to [Lookup by Name](/api-reference/lookup/lookup-by-name).

## Errors

| Status | Code | Cause |
|---|---|---|
| 400 | `INVALID_PARAMETER` | `url` contains null bytes or cannot be parsed as a valid domain |
| 400 | `INVALID_COUNTRY_CODE` | `country_iso_code` is not `AT`, `DE`, or `CH` |
| 401 | `AUTH_INVALID_KEY` | Invalid or missing API key |
| 429 | `RATE_LIMIT_EXCEEDED` | Rate limit hit |
