---
title: "Lookup by Name"
description: "Autocomplete-style company lookup by legal name."
openapi: "GET /v0/lookup/company"
---

Returns companies whose normalised legal name contains the `query` string, enabling partial and autocomplete-style lookups. For example, `"holzstudio sahm"` matches `"Holzstudio Sahm GmbH & Co. KG"`.

Use this endpoint to resolve a company name to a `company_id`, then fetch the full profile with [Get Company](/api-reference/company/get-company).

## Normalisation

The `query` parameter is normalised before matching using the same logic applied to the database — lowercased, with whitespace and punctuation stripped:

| Input | Normalised |
|---|---|
| `Holzstudio Sahm GmbH` | `holzstudiosahmgmbh` |
| `Muster & Co. KG` | `muster&cokg` |
| `Über-Tech AG` | `ubertechag` |

This means you can search with or without legal form suffixes, umlauts, punctuation, or case.

## Errors

| Status | Code | Cause |
|---|---|---|
| 400 | `INVALID_PARAMETER` | `query` is empty after normalisation, contains null bytes, or `country_iso_code` is not `AT`/`DE`/`CH` |
| 401 | `AUTH_INVALID_KEY` | Invalid or missing API key |
| 422 | — | `limit` exceeds 50 or `query` is shorter than 2 characters |
| 429 | `RATE_LIMIT_EXCEEDED` | Rate limit hit |
