title: "Lookup by Name"
description: "Autocomplete-style company lookup by legal name."
---

<Note>**Pricing:** €0.01 per successful call</Note>

```
GET https://api.proxdeal.com/v0/lookup/company
```

Returns companies whose normalised legal name contains the `query` string. Input is lowercased and stripped of whitespace and punctuation before matching, enabling partial and autocomplete-style lookups. For example, `"holzstudio sahm"` matches `"Holzstudio Sahm GmbH & Co. KG"`.

Use this endpoint to resolve a company name to a `company_id`, then fetch the full profile with [Get Company](/api-reference/company/get-company).

## Query parameters

<ParamField query="query" type="string" required>
  Company name to search. Minimum 2 characters. Partial matches are supported — the input is normalised before matching (lowercased, punctuation stripped).
</ParamField>

<ParamField query="country_iso_code" type="string">
  Restrict results to a single country: `AT`, `DE`, or `CH`.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Maximum results to return. Range: 1–50.
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Number of results to skip for pagination.
</ParamField>

## Example

<CodeGroup>
```bash cURL
curl "https://api.proxdeal.com/v0/lookup/company?query=Holzstudio+Sahm&country_iso_code=DE" \
  -H "Authorization: Bearer YOUR_API_KEY"
```
```python Python
import httpx

resp = httpx.get(
    "https://api.proxdeal.com/v0/lookup/company",
    params={"query": "Holzstudio Sahm", "country_iso_code": "DE"},
    headers={"Authorization": "Bearer YOUR_API_KEY"},
)
results = resp.json()["data"]
company_id = results[0]["company_id"]
```
```typescript TypeScript
const params = new URLSearchParams({ query: "Holzstudio Sahm", country_iso_code: "DE" });
const resp = await fetch(
  `https://api.proxdeal.com/v0/lookup/company?${params}`,
  { headers: { Authorization: "Bearer YOUR_API_KEY" } }
);
const { data, total_count } = await resp.json();
```
</CodeGroup>

## Normalisation

The `query` parameter is normalised before matching using the same logic applied to the database:

| Input | Normalised |
|---|---|
| `Holzstudio Sahm GmbH` | `holzstudiosahmgmbh` |
| `Muster & Co. KG` | `muster&cokg` |
| `Über-Tech AG` | `ubertechag` |

This means you can search with or without legal form suffixes, umlauts, punctuation, or case.

## Errors

| Status | Code | Cause |
|---|---|---|
| 400 | `INVALID_PARAMETER` | `query` is empty after normalisation, contains null bytes, or `country_iso_code` is not `AT`/`DE`/`CH` |
| 401 | `AUTH_INVALID_KEY` | Invalid or missing API key |
| 422 | — | `limit` exceeds 50 or `query` is shorter than 2 characters |
| 429 | `RATE_LIMIT_EXCEEDED` | Rate limit hit |
