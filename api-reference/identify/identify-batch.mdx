---
title: "Identify Batch"
description: "Submit a CSV or Excel file for bulk company identification."
openapi: "POST /v0/identify/batch"
---

Accepts a CSV or Excel file (up to 1,000 rows, 10 MB) as `multipart/form-data`. Maps columns to ProxDeal fields using one LLM call, then identifies each row's company in a background job. The submit call returns a `job_id` immediately — poll the status endpoint until the job completes. Optionally supply a `column_mapping` JSON string to skip the automatic column-mapping step.

## Poll for results

The submit endpoint above is asynchronous. Poll the companion endpoint with the returned `job_id` until it finishes:

```
GET https://api.proxdeal.com/v0/identify/batch/{job_id}
```

The response `status` is one of `pending`, `running`, `completed`, or `failed`. When `completed`, a `results` array holds one object per input row — each with `row_index`, `input_text`, `sdac_id` (resolved `company_id` or `null`), `company_legal_name`, `match_score` (0.0–1.0), and `match_status` (`"matched"` / `"unmatched"`).

<Tip>
  Poll every 5–10 seconds. Most jobs with under 200 rows complete in under 2 minutes. For 1,000-row files, allow up to 10 minutes.
</Tip>

## Errors

| Status | Code | Cause |
|---|---|---|
| 401 | `AUTH_INVALID_KEY` | Invalid or missing API key |
| 413 | — | File exceeds 10 MB |
| 422 | — | File format invalid or row count exceeds 1,000 |
| 429 | `RATE_LIMIT_EXCEEDED` | Rate limit hit |
| 503 | — | Identify service temporarily unavailable |
