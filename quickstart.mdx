## Prerequisites

- A ProxDeal account at [terminal.proxdeal.com](https://terminal.proxdeal.com)
- An active API key (create one in **API Settings → API Keys**)

## The three-call pattern

Most ProxDeal workflows follow the same pattern: **lookup → resolve → enrich**.

<Steps>
  <Step title="Look up a company by name or URL">
    Use the Lookup endpoint to resolve a name or website to a `company_id`. This is the entry point for almost every workflow.

    <CodeGroup>
    ```bash cURL
    curl "https://api.proxdeal.com/v0/lookup/company?query=Holzstudio+Sahm&country_iso_code=DE" \
      -H "Authorization: Bearer YOUR_API_KEY"
    ```
    ```python Python
    import httpx

    resp = httpx.get(
        "https://api.proxdeal.com/v0/lookup/company",
        params={"query": "Holzstudio Sahm", "country_iso_code": "DE"},
        headers={"Authorization": "Bearer YOUR_API_KEY"},
    )
    data = resp.json()
    company_id = data["data"][0]["company_id"]
    ```
    ```typescript TypeScript
    const resp = await fetch(
      "https://api.proxdeal.com/v0/lookup/company?query=Holzstudio+Sahm&country_iso_code=DE",
      { headers: { Authorization: "Bearer YOUR_API_KEY" } }
    );
    const data = await resp.json();
    const companyId = data.data[0].company_id;
    ```
    </CodeGroup>

    The response contains a list of matches. Each item includes a `company_id`, legal name, register number, and country.
  </Step>

  <Step title="Fetch the full company profile">
    Pass the `company_id` to the Company endpoint to get the full profile: address, industry classification, WZ2025 codes, representatives, website URL, and more.

    <CodeGroup>
    ```bash cURL
    curl "https://api.proxdeal.com/v0/company/de-0001z7cs0ej81r1w" \
      -H "Authorization: Bearer YOUR_API_KEY"
    ```
    ```python Python
    resp = httpx.get(
        f"https://api.proxdeal.com/v0/company/{company_id}",
        headers={"Authorization": "Bearer YOUR_API_KEY"},
    )
    company = resp.json()["data"]
    print(company["company_legal_name"], company["address_city"])
    ```
    ```typescript TypeScript
    const resp = await fetch(
      `https://api.proxdeal.com/v0/company/${companyId}`,
      { headers: { Authorization: "Bearer YOUR_API_KEY" } }
    );
    const { data } = await resp.json();
    ```
    </CodeGroup>
  </Step>

  <Step title="Enrich with financials">
    Pull the most recent annual financial report — revenue, profit, equity, employees, and 20+ more fields.

    <CodeGroup>
    ```bash cURL
    curl "https://api.proxdeal.com/v0/company/de-0001z7cs0ej81r1w/financials" \
      -H "Authorization: Bearer YOUR_API_KEY"
    ```
    ```python Python
    resp = httpx.get(
        f"https://api.proxdeal.com/v0/company/{company_id}/financials",
        headers={"Authorization": "Bearer YOUR_API_KEY"},
    )
    fin = resp.json()["data"]
    print(f"Revenue: €{fin['revenue']:,.0f}  Employees: {fin['employees']}")
    ```
    ```typescript TypeScript
    const resp = await fetch(
      `https://api.proxdeal.com/v0/company/${companyId}/financials`,
      { headers: { Authorization: "Bearer YOUR_API_KEY" } }
    );
    const { data } = await resp.json();
    ```
    </CodeGroup>
  </Step>
</Steps>

## Common use cases

| Use case | Endpoints |
|---|---|
| CRM enrichment | Lookup → Company → Financials |
| Ownership / compliance | Lookup → Shareholders (L1 or UBO) |
| Market screening | DataFinder Search |
| M&A target discovery | Longlist |
| Website-to-company resolution | Lookup by URL |
| Bulk list matching | Identify Batch |

## Response envelope

All successful responses share the same envelope:

```json
{
  "status": "success",
  "data": { ... },
  "message": "...",
  "timestamp": "2026-01-15T10:30:00Z",
  "request_id": "req_abc123def456"
}
```

Save the `request_id` — include it in any support request to help us trace the call.
